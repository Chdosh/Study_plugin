## 4. 问题详情

### BUG-001：浮窗收起后保留展开态空白遮挡

* 严重度：P1
* 发生位置：学习浮窗
* 前置条件：已开始学习，浮窗处于可展开状态。
* 复现步骤：
  1. 点击当前重点任务的“开始学习”。
  2. 点击浮窗右侧“展开”按钮。
  3. 点击浮窗右侧“收起”按钮。
* 预期结果：浮窗收起后恢复为约 380×48 的紧凑条，只显示任务、计时和控制按钮。
* 实际结果：浮窗内容区消失，但窗口仍保持约 380×300，高度不回收，留下大面积空白遮挡主窗口。
* 复现次数：2 次。
* 是否稳定复现：是。
* 是否必须重启：否，但等待 5 秒不会自动恢复，只能再次展开或结束进程缓解。
* 是否阻断后续测试：部分阻断。浮窗遮挡主窗口中部，影响主窗口操作和观察。
* 截图路径：
  * `docs/test-report-v1/evidence/smoke/10-float-collapsed.png`
  * `docs/test-report-v1/evidence/smoke/11-float-collapse-blank-after-wait.png`
  * `docs/test-report-v1/evidence/smoke/13-float-collapse-blank-repro2.png`
* 建议的正确产品行为：收起时同步调整窗口高度，并隐藏展开内容容器；收起后的浮窗不应覆盖主窗口大片区域。
* 修复后的验收标准：连续执行“展开 -> 收起”两次后，浮窗截图高度稳定回到紧凑态，主窗口中部不被空白浮层遮挡。

### BUG-002：暂停恢复后学习计时被重置

* 严重度：P1
* 发生位置：学习浮窗、主窗口学习页、学习结算页
* 前置条件：已开始学习并运行约 1 分钟。
* 复现步骤：
  1. 从浮窗点击“暂停”，主窗口和浮窗显示 01:00。
  2. 从浮窗点击“继续”。
  3. 再从主窗口点击“暂停学习”，主窗口和浮窗显示 01:00。
  4. 从主窗口点击“恢复学习”。
* 预期结果：恢复学习后累计有效学习时长继续保留，不应回到 00:00/00:01；结算时长应能反映本次学习累计时间。
* 实际结果：浮窗恢复后显示 00:00，主窗口恢复后显示 00:01；后续结算页“本次时长”只显示 1 分钟。
* 复现次数：2 次，分别从浮窗恢复和主窗口恢复触发。
* 是否稳定复现：是。
* 是否必须重启：否。
* 是否阻断后续测试：部分阻断。后续复盘和统计会基于错误时长，影响学习结果可信度。
* 截图路径：
  * `docs/test-report-v1/evidence/smoke/14-float-paused.png`
  * `docs/test-report-v1/evidence/smoke/16-float-resumed.png`
  * `docs/test-report-v1/evidence/smoke/18-main-paused-from-main.png`
  * `docs/test-report-v1/evidence/smoke/20-main-resumed-from-main.png`
  * `docs/test-report-v1/evidence/smoke/24-main-after-end-click.png`
* 建议的正确产品行为：暂停应冻结累计有效时长；恢复应从累计时长继续显示；结算页应使用同一累计时长来源。
* 修复后的验收标准：学习 60 秒后暂停，再恢复 10 秒，主窗口、浮窗和结算页应显示约 1 分 10 秒或一致的分钟级累计结果。

### BUG-003：保存结算后复盘页仍为空，复盘/调整流程无法继续

* 严重度：P1
* 发生位置：学习结算页、复盘页
* 前置条件：已开始并结束一个学习块，进入学习结算页。
* 复现步骤：
  1. 在学习页点击“结束学习”。
  2. 保持默认“完成本块”。
  3. 点击“保存结算并进入复盘”。
* 预期结果：进入复盘页后应显示刚刚完成的学习块数据、结算结果，或至少提供可继续生成复盘/调整建议的明确操作。
* 实际结果：进入复盘导航后页面显示“还没有复盘 / 完成或跳过学习块后再生成复盘”，与刚刚完成并保存学习块的动作矛盾；未出现复盘结果或调整建议。
* 复现次数：1 次。
* 是否稳定复现：未重复触发，避免重复提交学习结算数据。
* 是否必须重启：否。
* 是否阻断后续测试：是。AI 复盘、计划调整、确认/拒绝调整的后续核心流程无法继续。
* 截图路径：
  * `docs/test-report-v1/evidence/smoke/24-main-after-end-click.png`
  * `docs/test-report-v1/evidence/smoke/25-main-after-save-settlement.png`
* 建议的正确产品行为：保存结算后，复盘页应读取本次 session/block 结果；如果 AI 复盘不可用，应显示明确的本地结算摘要和下一步动作，而不是空复盘。
* 修复后的验收标准：完成学习块并保存结算后，复盘页能看到本次学习记录、完成状态、时长和可执行的下一步；不会出现“还没有复盘”的空状态。

### BUG-004：无活跃会话时残留浮窗，关闭主窗口后浮窗无法普通关闭

* 严重度：P1
* 发生位置：应用启动、学习浮窗、应用关闭
* 前置条件：已保存学习结算并关闭/重启应用。
* 复现步骤：
  1. 完成学习结算并进入复盘。
  2. 关闭主窗口。
  3. 重新执行 `npm.cmd run dev` 启动应用。
  4. 检查主窗口和浮窗。
  5. 再关闭主窗口，观察剩余窗口。
* 预期结果：无活跃学习会话时不应显示学习浮窗；关闭主窗口后应用应完全关闭，或浮窗应提供明确可关闭/返回主窗口的控制。
* 实际结果：重启后主窗口显示今日页和“开始学习”，但同时出现没有可用控制文本的学习浮窗；关闭主窗口后只剩“学习浮窗”窗口，Alt+F4 未关闭该浮窗，必须停止开发启动进程才能清理。
* 复现次数：2 次观察到启动/重启后浮窗残留；关闭主窗口后残留浮窗 1 次。
* 是否稳定复现：是，重启后可见；关闭残留浮窗未继续反复尝试。
* 是否必须重启：清理残留浮窗需要停止启动进程；普通窗口关闭无法恢复。
* 是否阻断后续测试：部分阻断。后续启动状态、主/浮窗同步和关闭行为无法可信测试。
* 截图路径：
  * `docs/test-report-v1/evidence/smoke/01-main-initial.png`
  * `docs/test-report-v1/evidence/smoke/27-restart-main.png`
  * `docs/test-report-v1/evidence/smoke/28-restart-float.png`
* 建议的正确产品行为：浮窗生命周期必须绑定活跃学习会话；无会话时不创建浮窗。关闭主窗口时如果没有活跃会话，应关闭整个应用；如果有会话，应明确提示或让浮窗具备可恢复主窗口/结束会话/关闭的可见控制。
* 修复后的验收标准：保存结算后重启应用只出现主窗口，不出现无控制浮窗；关闭主窗口后无残留 Electron 窗口。

## 5. 哪些后续测试被阻断

- AI 复盘内容生成、复盘结果展示、计划调整建议确认/拒绝被 BUG-003 阻断。
- 会话时长、专注统计、复盘评分、完成率统计的可信验证被 BUG-002 阻断。
- 浮窗长期可用性、窗口关闭/重启干净状态、无会话启动状态被 BUG-001 和 BUG-004 阻断。

## 6. 建议的修复顺序

1. 先修 BUG-004，确保浮窗生命周期和应用关闭行为可控，否则每次测试都会带入残留状态。
2. 再修 BUG-001，避免浮窗遮挡主窗口并干扰后续操作。
3. 再修 BUG-002，保证会话计时是可信数据源。
4. 最后修 BUG-003，打通结算后的复盘/调整入口。

## 7. 修复完成后应重新执行的测试

1. 启动应用，确认无活跃会话时只有主窗口，没有学习浮窗。
2. 点击“开始学习”，确认只创建一个浮窗、一个学习会话。
3. 对浮窗执行单击、双击、展开、收起，确认无空白遮挡和重复窗口。
4. 分别从浮窗和主窗口执行暂停/恢复，确认计时不归零。
5. 结束学习并保存结算，确认复盘页显示本次学习记录和下一步动作。
6. 关闭并重启应用，确认状态持久化正确且无残留浮窗。
