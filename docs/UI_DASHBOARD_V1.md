## 设计目标
- 将现有“学习工作台”从功能堆叠改为清晰的执行控制台。
- 保持本地优先、AI 建议需确认、中文 UI 的产品约束。
- 第一屏聚焦一个主流程：导入计划 -> 生成草稿 -> 确认计划 -> 执行学习 -> 复盘。
- 使用三栏工作台布局承载当前阶段最重要的信息，而不是一次性展示所有数据。

## 信息架构

### 全局结构
- 左侧全局导航：学习工作台、任务清单、复盘、设置。
- 顶部全局状态栏：当前页面标题、流程状态、API Key 状态、刷新入口。
- 主区域三栏：
  - 左栏：输入与控制。
  - 中栏：今日计划和执行时间线。
  - 右栏：完成度、历史版本、当前专注块。

### 学习工作台
- 左栏输入与控制：
  - 粘贴计划输入框。
  - 来源选择：ChatGPT、Codex、手动。
  - 提示词模式：基础、标准、考核、恢复。
  - 主操作：导入到今日上下文、生成今日草稿。
  - 参数：未完成任务数、学习块长度、可用时间窗。
- 中栏今日计划：
  - 顶部 stepper 显示 5 阶段流程。
  - 今日计划状态、日期、草稿/已确认标签。
  - 10 分钟学习块列表。
  - 每块显示目标、动作、产出、验收、降级动作、状态和操作。
- 右栏状态概览：
  - 完成度和历史版本数。
  - 当前专注块倒计时。
  - 今日历史版本时间线。
  - 本地监控边界提示：只记录前台应用、窗口标题、离开时间。

### 任务清单
- 独立页面，面向整理和调整任务，而不是执行。
- 卡片式任务列表，包含标题、描述、难度、状态和操作菜单。
- 筛选入口：全部、待安排、进行中、已完成。

### 复盘页面
- 左侧为 AI 复盘草稿：完成度、专注度、总结、下一步动作。
- 右侧为人工确认区：用户确认后才进入后续计划调整。
- 强调复盘建议不会自动覆盖已确认计划。

## 视觉系统
- 风格：安静、清晰、工作台式，不做营销页。
- 主色：深墨绿导航 + 青绿色主操作。
- 辅色：琥珀表示草稿/提醒，紫色表示进行中，红色表示跳过/风险。
- 背景：低饱和浅灰绿，减少长时间学习时的视觉疲劳。
- 圆角：统一 8px，符合当前 CSS 和桌面工具质感。
- 信息密度：比当前界面更有层次，但保留 B 端操作效率。

## 交互原则
- AI 生成内容默认是草稿，确认按钮必须显眼。
- 空状态需要引导下一步，但不占用过多第一屏空间。
- 高级参数默认弱化展示，避免干扰导入和生成主流程。
- 计划历史保留版本感，方便比较多次生成结果。
- 当前专注块始终在右栏可见，避免执行阶段迷失。
