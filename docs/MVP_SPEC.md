# MVP Spec

状态：ACTIVE  
用途：当前版本的实现规格和验收依据。  
约束：本文件必须遵守 `docs/PRODUCT_TRUTH.md`。Product Truth 定义长期产品事实；本文件只定义当前 MVP 的默认流程、策略、AI 调用边界和验收标准。

## 1. 当前 MVP 范围

当前 MVP 关注一个可用、可恢复的学习闭环：

1. 用户创建新目标，或继续已有目标。
2. 当前目标信息不足时，AI 进行简短主动访谈；已有目标和上下文足够时直接恢复进度。
3. 用户确认或修改简短“目标理解”，也可以在信息足够时直接生成计划。
4. 系统先建立阶段结构，再生成近期重点和当前可执行任务。
5. 当前版本默认只详细展开当前学习日，不提前生成全部详细任务。
6. 用户可以在 Today 查看今日进度、知识库和计划状态；学习操作集中在 Study 页面。
7. 用户在 Study 页面围绕当前主任务开始、暂停、恢复、提问、记录进度、最终提交并获得评价。
8. 在主任务完成或阶段结束后，用户可以进入 Review 生成相应复盘，并确认是否延续、缩小、拆分、延后或放弃未完成任务。

分层计划默认包括长期阶段、近期重点和未来几天的粗略安排，具体展开范围根据目标和时间条件决定。

## 2. 当前业务结构

这是当前 MVP 的业务结构，用于统一实现语义，不代表数据库表名或永久字段命名。

```text
LearningGoal
└── RoadmapStage
    └── DailyGuide
        └── Task
            ├── Action
            ├── Checkpoint
            ├── FocusSession
            ├── Submission
            └── Evaluation
```

计划和学习状态必须结构化保存，不能只存在于聊天记录或模型上下文中。

当前实现中，Action 表示主任务内的执行步骤；Checkpoint 表示本地完成检查或完成标准。它们服务于执行引导，不是独立的正式评价单位。

## 3. 当前流程

1. 用户创建新目标或当前目标信息不足时，AI 发起简短主动访谈。
2. 信息基本足够时，AI 输出目标理解。
3. 用户确认目标理解，修改目标理解，或直接要求先生成计划。
4. 系统建立阶段结构，再生成近期重点和当前可执行任务；当前 MVP 默认详细展开今天，未来内容保持概括。
5. 用户在 Today 查看、修改或要求重新生成今日主任务；首次或重大计划调整仍需用户确认。
6. 系统一次只完整展示当前主任务，包括目标、范围、预计时间、行动、成果和完成标准。
7. 其他当日主任务默认折叠或简要展示。用户可以预览，也可以主动切换；切换前保存当前任务进度。
8. 用户围绕当前主任务提问、保存本地进度、提交最终结果并获得评价。
9. AI 建议必须经用户确认后才调整计划。

用户提交主任务最终结果后，必须先评价。当前主流程不固定调用第二次 AI 决策；本地状态机根据评价结果决定完成、继续修改、保存进度或进入下一任务。

## 4. 规划与展开策略

长期阶段只描述阶段结构和方向，不生成所有详细学习步骤。

近期重点描述未来几天的粗略安排，不提前展开所有任务细节。

今日任务通常为 1 到 3 个，优先保证任务完整性和执行清晰度。可用时间较长或任务较短时可以增加，但不为了满足数量强行拆分任务。

不再预先生成固定 10 分钟 Time Block。每个主任务应提供最短、常规和最长预计时间范围。当前 MVP 中计时器绑定主任务：开始主任务时开始计时，暂停/恢复只改变当前主任务计时状态，主任务提交评价通过后同步结束当前 Focus Session；不提供独立于主任务完成的“结束计时/结束学习”主操作。一个主任务后续仍可通过恢复进度继续。

Action 和 Checkpoint 只做本地执行引导和进度记录，不分别触发 AI 评价。后续步骤可根据用户实际表现动态生成；未到达的节点可以保持折叠或尚未生成。

不得提前生成全部详细步骤，也不得让 AI 未经确认永久修改已确认计划。

## 5. AI 调用边界

AI 可以在以下节点调用：

* 目标信息不足时，生成简短追问。
* 信息足够时，生成目标理解。
* 用户确认后，生成或更新分层计划和当前可执行任务。
* 用户围绕当前任务提问时，回答问题并保持原任务上下文。
* 用户提交主任务结果后，按任务类型生成结构化评价。
* 主任务完成或阶段结束后，按用户操作生成复盘建议。

AI 不得直接标记任务完成，不得绕过运行时校验，不得未经用户确认永久修改计划、画像或持久状态。

Focus Session 的开始、暂停、恢复和超时不触发 AI。当前 MVP 不提供独立结束 Focus Session 的页面操作；主任务提交评价通过时同步结束当前 Focus Session。Action 完成、Checkpoint 记录和时间流逝也不单独触发 AI 评价。

## 6. 问题分支规则

临时提问必须绑定当前目标、主任务和执行位置，不能替代学习主线。

每个问题必须记录所属目标、主任务、执行位置、处理状态、创建时间和解决摘要。具体字段名以 TypeScript 类型、schema 或数据模型文档为准。

打开问题不能改变当前主任务。问题解决后，应生成问题摘要、标记为已解决、清除当前问题分支，并返回原主任务和原执行位置。

## 7. 信息架构

当前 MVP 推荐采用以下主导航；后续允许在不破坏核心能力的前提下调整名称、层级或合并页面：

* Today
* Study
* Review
* Settings

Today 聚焦当前学习日、进度和知识库，不承载开始/暂停/提交等学习操作。Study 聚焦当前任务执行、行动步骤、开始/暂停/恢复、提问和提交。Review 聚焦历史、复盘和后续调整，不能作为学习流程中强制弹出的独立结算页，也不提供开始/继续学习入口。Settings 聚焦模型、隐私和本地配置，不提供浮窗、计时提醒或自动跳转复盘等独立流程控制。

当前 MVP 不创建独立学习浮窗，不通过托盘菜单启动学习或生成复盘，也不允许主进程强制导航到某个页面。页面间切换由全局导航承载，页面内部只保留本页职责内的操作。

AI tutor 应作为上下文助手出现，而不是孤立的通用聊天页。

## 8. 当前 UI 与交互策略

当前 MVP 默认一次只完整展示当前任务：

* 当前任务完整展开。
* 其他任务简略显示。
* 当前任务显示目标、范围、预计时长、行动、成果和完成标准。
* 用户可以主动切换任务；切换前保存当前任务进度。
* 当前任务的提问、提交和评价入口应与任务上下文绑定。

聚焦通过默认展示、折叠和提示实现，不通过禁止用户操作实现。

## 9. 本地恢复要求

应用重启后至少应能恢复：

* 当前学习目标和阶段位置。
* 当前学习日和当前主任务。
* 当前任务的行动步骤进度。
* 正在进行或暂停中的 Focus Session 状态。
* 已保存的问题分支、解决摘要、提交和评价。
* 未完成任务的继续、缩小、拆分、延后或放弃建议。

## 10. MVP 验收

MVP 至少应能验证：

* 新目标或信息不足时，能通过主动访谈形成可确认的目标理解。
* 已有目标和上下文足够时，能直接恢复进度。
* 能建立阶段结构、近期重点和当前学习日执行任务。
* 当前学习日默认只完整展开当前主任务。
* 其他当日主任务可以预览或主动切换，切换前保存当前进度。
* 连续提问不会丢失主线。
* Focus Session 的开始、暂停、恢复和超时不会触发 AI。
* Action 和 Checkpoint 只做本地记录，不触发 AI 评价。
* 主任务最终提交后可以获得结构化评价。
* 本地状态机能根据评价结果进入完成、继续修改、保存进度或下一任务。
* 在主任务完成或阶段结束后，可以由用户进入 Review 生成相应复盘。
* 重启后恢复当前学习位置。

## 11. 当前版本 Non-Goals

当前 MVP 暂不实现：

* 可拖动思维导图或复杂计划画布。
* 复杂自治多 Agent 协作框架。
* 复杂知识图谱。
* 在核心任务、Session 和 Review 数据稳定前实现 RAG。
* 没有隐私和冲突解决设计的云同步。
* 多设备实时同步。
