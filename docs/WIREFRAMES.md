# 低保真线框 V1

日期：2026-06-20
依据：`docs/PRODUCT_SCOPE_V1.md`、`docs/USER_FLOWS.md`。
范围：定义页面结构和状态，不讨论具体颜色、阴影、动画、插画或高保真视觉。

## 0. 通用布局约定

```text
┌────────────────────────────────────────────────────────────┐
│ 顶部状态区：页面标题 / 当前系统状态 / 全局反馈              │
├──────────┬─────────────────────────────────────┬───────────┤
│ 一级导航 │ 主内容区                              │ 上下文区   │
│ 今日     │ 根据页面职责展示核心任务              │ AI教师/详情 │
│ 计划     │                                     │ 可为空     │
│ 学习     │                                     │           │
│ 知识     │                                     │           │
│ 复盘     │                                     │           │
│ 设置     │                                     │           │
└──────────┴─────────────────────────────────────┴───────────┘
```

- 一级导航固定：今日、计划、学习、知识、复盘、设置。
- 每页只保留一个最强主操作。
- AI 教师面板优先放在上下文区；学习页可以常驻，其他页面可折叠。
- 重要确认使用弹窗或确认面板，不用普通提示文本代替。

## 1. 今日

### 页面标题

今日

### 主操作

根据当前状态动态显示一个主按钮：

- 去设置
- 去计划页创建/导入
- 去计划页生成今日草稿
- 去计划页确认草稿
- 开始学习
- 继续学习
- 去复盘

### 次要操作

- 查看今日计划摘要
- 查看风险提示
- 跳转计划页
- 跳转复盘页
- 跳转设置页

### 内容区域

```text
┌ 今日 ───────────────────────────────────────────────┐
│ 当前状态摘要：API / 任务 / 今日计划 / Session        │
│                                                      │
│ ┌ 下一步建议 ─────────────────────────────────────┐ │
│ │ 当前应做：开始学习 / 去计划 / 去设置             │ │
│ │ 原因：已有确认计划，下一块尚未开始               │ │
│ │ [主操作按钮]                                    │ │
│ └────────────────────────────────────────────────┘ │
│                                                      │
│ ┌ 今日确认计划摘要 ───────────────────────────────┐ │
│ │ 当前推荐块：目标 / 动作 / 验收                   │ │
│ │ 下一块：标题 / 预计分钟                         │ │
│ │ 进度：已完成块 / 总块数 / 未完成块               │ │
│ └────────────────────────────────────────────────┘ │
│                                                      │
│ ┌ 风险提示 ───────────────────────────────────────┐ │
│ │ 计划未确认 / 缺 API Key / 无任务 / 有跳过记录     │ │
│ └────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘

右侧上下文区：
┌ AI教师 ────────────────┐
│ 解释为什么推荐当前动作 │
│ 提醒计划或配置风险     │
│ 不生成完整计划         │
└────────────────────────┘
```

### 空状态

- 无任务：提示“还没有可学习任务”，主操作为“去计划页创建或导入”。
- 无今日计划但有任务：提示“已有任务，尚未生成今日计划”，主操作为“生成今日草稿”。

### 加载状态

- 加载今日状态、计划摘要、active session。
- 主操作按钮进入禁用或骨架占位，避免重复创建 session。

### 错误状态

- 数据库读取失败：显示重试入口。
- 今日计划冲突：提示存在多个 confirmed plan，主操作跳转计划页处理。
- active session 恢复失败：提示进入学习页处理异常 session。

### AI 不可用状态

- 今日页仍可显示本地状态。
- AI 教师面板显示“AI 不可用”，并提供“去设置”入口。
- 不阻塞“开始学习”“继续学习”等本地动作。

## 2. 计划

### 页面标题

计划

### 主操作

- 未解析输入时：解析学习输入
- 已有任务选择时：生成今日草稿
- 已有草稿时：确认草稿

### 次要操作

- 保存草稿输入
- 重新解析
- 排除任务
- 重新生成草稿
- 放弃草稿
- 查看计划历史

### 内容区域

```text
┌ 计划 ───────────────────────────────────────────────┐
│ 步骤条：输入 -> 解析 -> 选择任务 -> 生成草稿 -> 确认 │
│                                                      │
│ ┌ 输入区 ─────────────────────────────────────────┐ │
│ │ 粘贴/手动输入学习目标或计划                     │ │
│ │ 来源：ChatGPT / Codex / 手动                    │ │
│ │ 提示词档位                                      │ │
│ │ [解析学习输入]                                  │ │
│ └────────────────────────────────────────────────┘ │
│                                                      │
│ ┌ 解析结果检查 ───────────────────────────────────┐ │
│ │ 目标列表                                        │ │
│ │ 任务列表：勾选 / 标题 / 估时 / 难度 / 验收 / 依赖│ │
│ │ [生成今日草稿]                                  │ │
│ └────────────────────────────────────────────────┘ │
│                                                      │
│ ┌ 草稿计划审查 ───────────────────────────────────┐ │
│ │ 草稿状态：AI 建议，未确认                       │ │
│ │ 计划块列表：目标 / 动作 / 输出 / 验收 / 降级动作 │ │
│ │ [确认草稿] [重新生成] [放弃草稿]                 │ │
│ └────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘

右侧上下文区：
┌ AI教师 ─────────────────────────┐
│ 解释解析结果和任务拆分           │
│ 说明草稿计划的依据               │
│ 提醒 AI 不会自动覆盖已确认计划    │
└──────────────────────────────────┘

抽屉：
- 任务详情
- 计划历史详情
- AI 调整建议来源
```

### 空状态

- 没有输入：显示输入引导和示例占位，不展示任务表。
- 没有可规划任务：提示先导入或创建任务。
- 没有草稿：任务选择区显示“选择任务后生成草稿”。

### 加载状态

- 解析中：输入区保持只读，结果区显示解析中。
- 生成计划中：草稿区显示生成中，不影响已有 confirmed plan。
- 确认中：确认按钮禁用，防止重复确认。

### 错误状态

- 输入为空：提示不能解析空内容。
- AI 输出格式错误：提示未写入正式计划，可重试或修改提示词。
- 已有 confirmed plan：确认新草稿前显示替换/归档确认。
- 数据保存失败：草稿状态不应显示为已确认。

### AI 不可用状态

- 显示 API Key 缺失或连接失败。
- 允许手动创建/整理任务。
- 禁用 AI 解析、生成草稿和 AI 教师说明。
- 提供去设置入口。

### 空状态

- 无 confirmed plan：提示回计划页确认计划。
- 无可执行块：提示今日计划已完成或计划异常，提供去复盘/计划页入口。

### 加载状态

- 加载当前 block 和 active session。
- 恢复 session 时显示恢复中。
- 保存备注时显示保存中，但不阻塞学习内容阅读。

### 错误状态

- block 不存在或状态冲突：提示返回计划页修复。
- session 创建失败：提示重试，block 不应变为 active。
- 监控不可用：session 可继续，监控状态显示不可用。
- 保存完成失败：不进入完成状态。

### AI 不可用状态

- 学习主流程照常可用。
- AI 教师面板显示不可用和去设置入口。
- 不影响备注、完成、跳过、暂停。

### 加载状态

- 加载知识列表。
- 打开详情抽屉时加载来源引用。
- 保存条目时显示保存中。

### 错误状态

- 条目保存失败：不显示已保存。
- 来源记录不存在：允许作为手动知识显示，但标记来源缺失。
- 删除或归档失败：保留原条目。

### AI 不可用状态

- 列表、搜索、手动保存照常可用。
- AI 摘要、标题建议、标签建议不可用。
- 不自动收集窗口标题或私密内容。

## 8. AI 计划调整确认

### 页面标题

AI 计划调整确认

### 建议容器

弹窗、确认页或复盘页内的专用确认面板。建议在涉及多条调整时使用专用确认面板。

### 主操作

确认并生成新草稿

### 次要操作

- 拒绝建议
- 稍后处理
- 查看当前计划
- 返回复盘

### 内容区域

```text
┌ AI 计划调整确认 ────────────────────────────────────┐
│ 提示：以下为 AI 建议，尚未应用，不会覆盖 confirmed plan│
│                                                      │
│ ┌ 当前计划摘要 ───────────────────────────────────┐ │
│ │ 当前 confirmed plan 日期 / 块数 / 未完成任务      │ │
│ └────────────────────────────────────────────────┘ │
│                                                      │
│ ┌ 调整建议列表 ───────────────────────────────────┐ │
│ │ 建议 1：拆分任务                                 │ │
│ │ 原因：实际耗时超出估计                           │ │
│ │ 影响：任务 A / 明日计划                           │ │
│ │ 差异：原块 -> 新块                                │ │
│ │ 冲突状态：无 / 有冲突                             │ │
│ │                                                   │ │
│ │ 建议 2：降低难度或增加前置任务                   │ │
│ └────────────────────────────────────────────────┘ │
│                                                      │
│ ┌ 应用结果说明 ───────────────────────────────────┐ │
│ │ 确认后只生成新的 draft plan 或 plan version       │ │
│ │ 仍需在计划页审查并确认                           │ │
│ └────────────────────────────────────────────────┘ │
│                                                      │
│ [确认并生成新草稿] [拒绝建议] [稍后处理]             │
└──────────────────────────────────────────────────────┘
```

### 空状态

- 没有 pending proposal：显示“没有待确认调整建议”，返回复盘。

### 加载状态

- 加载 proposal 和当前 confirmed plan。
- 生成新草稿中。
- 标记 rejected 或 pending 中。

### 错误状态

- 原 confirmed plan 已变化：提示建议可能过期，需要重新生成或重新审查。
- 涉及任务已完成、删除或状态冲突：显示冲突，不允许直接确认。
- 新草稿生成失败：proposal 保持 pending 或 failed，confirmed plan 不变。

### AI 不可用状态

- 已生成的 pending proposal 可以继续审查。
- 无法重新生成解释或补充建议。
- 确认动作只基于已保存 proposal 和本地数据。

## 9. 线框结论

V1 线框应服务完整闭环：

```text
今日判断下一步
-> 计划生成并确认
-> 学习执行
-> 学习结束结算
-> 复盘
-> AI 计划调整确认
-> 回到计划形成新草稿
-> 知识沉淀
```

任何页面如果无法支持这个闭环，应降级为 Future 或从主路径中移除。
