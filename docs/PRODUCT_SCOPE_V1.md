# 产品范围 V1

日期：2026-06-20
依据：`AGENTS.md`、`docs/CURRENT_PRODUCT_AUDIT.md`。
约束：本文件定义 V1 产品范围，不定义具体配色，不引入超出 `AGENTS.md` 的新功能，不代表已经完成实现。

## 0. V1 产品定位

V1 是一个本地优先的 Windows 桌面 AI 学习管家，目标不是做完整知识管理平台，而是把核心学习闭环做稳：

```text
导入/创建目标
-> 生成可执行计划
-> 确认计划
-> 开始学习
-> 记录执行
-> 完成/跳过/调整
-> 复盘
-> 生成下一步计划建议
```

V1 的一级导航固定为：

- 今日
- 计划
- 学习
- 知识
- 复盘
- 设置

每个页面只承担清晰的产品职责。不要让一个页面同时承担导入、规划、执行、复盘、设置等过多目标。

## 1. 页面范围

## 1.1 今日

### 核心用户目标

回答用户打开软件后的第一个问题：

- 我现在应该学什么？
- 今天最重要的学习动作是什么？
- 我是否已经有确认计划？
- 下一步应该点击哪里？

### 必须存在的功能

- 显示今日状态：
  - 是否已配置 DeepSeek。
  - 是否已有任务。
  - 是否已有今日草稿计划。
  - 是否已有今日确认计划。
  - 是否存在进行中的学习 session。
- 显示今日主行动作，只保留一个最强主按钮：
  - 缺 API Key：去设置。
  - 无任务：去计划页导入/创建任务。
  - 有任务无计划：去计划页生成今日草稿。
  - 有草稿未确认：去计划页确认草稿。
  - 有确认计划无进行中 session：开始学习。
  - 有进行中 session：继续学习。
  - 今日已完成主要计划：去复盘。
- 展示今日确认计划摘要：
  - 当前推荐学习块。
  - 下一个学习块。
  - 今日完成进度。
  - 未完成块数量。
- 展示简单风险提示：
  - 计划未确认。
  - API Key 缺失。
  - 今日没有可学习任务。
  - 存在跳过/延期记录。
- 入口跳转：
  - 去计划页。
  - 去学习页。
  - 去复盘页。
  - 去设置页。

### 暂时不做的功能

- 不在今日页提供大段计划导入文本框。
- 不在今日页编辑任务细节。
- 不在今日页编辑 prompt profile。
- 不在今日页展示完整历史计划 diff。
- 不在今日页做复杂数据报表。
- 不在今日页展示知识库或 RAG 搜索结果。

## 1.2 计划

### 核心用户目标

把外部学习意图变成本地可执行计划，并让用户确认 AI 生成的草稿。

### 必须存在的功能

- 导入或创建学习输入：
  - 粘贴 ChatGPT/Codex/手动学习计划。
  - 选择来源。
  - 选择提示词档位。
- AI 解析导入内容：
  - 创建目标。
  - 创建任务。
  - 创建任务依赖。
  - 生成估时、难度、验收标准。
- 展示解析结果供用户检查：
  - 新增任务数量。
  - 任务标题。
  - 估计分钟数。
  - 难度。
  - 验收标准。
  - 依赖关系摘要。
- 任务进入计划前的最小整理：
  - 勾选或排除本次要进入今日计划的任务。
  - 标记任务是否仍待安排、已完成、跳过。
- 生成今日草稿计划：
  - 使用选中的未完成任务。
  - 按默认学习块分钟数生成计划块。
  - 每块包含目标、动作、预期输出、验收标准、难度、材料、降级动作。
- 草稿计划审查：
  - 展示计划块列表。
  - 展示草稿状态。
  - 展示 AI 输出只是建议。
  - 支持确认草稿。
  - 支持放弃草稿或重新生成。
- 计划历史：
  - 展示同一天的草稿/确认计划。
  - 明确哪个是当前确认计划。
  - 保留计划版本记录。

### 暂时不做的功能

- 不做拖拽式长期排期。
- 不做复杂甘特图。
- 不做自动抓取 ChatGPT/Codex 网页内容。
- 不做多用户协作。
- 不做完整课程系统。
- 不做复杂知识图谱。
- 不让 AI 自动覆盖已确认计划。

## 1.3 学习

### 核心用户目标

进入一个明确的学习块，知道本块要做什么、做到什么程度、如何结束或跳过。

### 必须存在的功能

- 显示当前学习块：
  - 目标。
  - 动作。
  - 预期输出。
  - 验收标准。
  - 所需材料。
  - 难度。
  - 太难时的降级动作。
- 学习 session 控制：
  - 开始。
  - 暂停。
  - 继续。
  - 完成。
  - 跳过。
  - 记录备注。
- 监控状态可见：
  - 当前是否正在记录前台应用。
  - 当前 session 开始时间。
  - 监控范围说明：只记录前台应用名、窗口标题、切换事件。
- 完成学习块时区分：
  - 完成此学习块。
  - 是否完成整个任务。
  - 部分完成或需要继续。
- 跳过学习块时记录原因。
- 学习完成后回到今日或复盘。

### 暂时不做的功能

- 不做截图监控。
- 不做键盘记录。
- 不做浏览器历史读取。
- 不做强制锁屏。
- 不做复杂倒计时动画。
- 不做实时 AI 语音教练。
- 不做完整 IDE 或资料阅读器。

## 1.4 知识

### 核心用户目标

沉淀学习过程中的重要资料、问题、错误和总结，为后续 RAG/知识库做准备。

### 必须存在的功能

V1 只做轻量知识入口，不做完整知识库系统：

- 展示可沉淀的知识来源类型：
  - 导入计划原文。
  - 学习 session 备注。
  - 复盘总结。
  - 用户记录的问题。
  - 用户记录的错误。
- 支持从学习或复盘中保存简单笔记。
- 每条知识记录至少应保留：
  - 来源。
  - 关联任务或 session。
  - 创建时间。
  - 用户可读内容。
- 支持基础列表查看。
- 支持基础关键词搜索可以作为后续迭代目标，但 V1 不要求向量检索。

### 暂时不做的功能

- 不做 LangChain/LlamaIndex。
- 不做向量数据库。
- 不做复杂知识图谱。
- 不做自动网页抓取。
- 不做全文资料管理器。
- 不做自动把所有窗口标题或私密内容写入知识库。
- 不做跨设备同步。

### Future

- Future：SQLite FTS5 关键词搜索。
- Future：文档导入与 chunking。
- Future：sqlite-vec 或 LanceDB 向量检索。
- Future：RAG 检索日志和召回质量评估。
- Future：长期用户画像和记忆摘要。

## 1.5 复盘

### 核心用户目标

理解今天执行得怎么样，并把复盘结果转化成下一步计划建议。

### 必须存在的功能

- 展示今日执行数据：
  - 计划块完成情况。
  - session 时长。
  - 跳过/延期原因。
  - 前台应用切换摘要。
- 生成 AI 复盘：
  - 完成度评分。
  - 专注度评分。
  - 难度匹配判断。
  - 今日问题总结。
  - 下一步动作建议。
- 复盘结果持久化：
  - 可查看历史复盘。
  - 当前日期复盘刷新后仍可见。
- 计划调整建议：
  - AI 可以提出拆分任务、降低难度、增加前置任务、延期、增加练习。
  - 建议必须可审查。
  - 用户必须确认或拒绝。
  - 确认后才进入计划页形成新草稿或新版本。

### 暂时不做的功能

- 不自动覆盖已确认计划。
- 不自动标记任务完成。
- 不做复杂仪表盘。
- 不做社交排名。
- 不做长期趋势图表，除非核心闭环稳定。
- 不把原始窗口标题直接发送给 AI，除非后续明确设计披露和摘要策略。

## 1.6 设置

### 核心用户目标

配置本地应用运行所需的模型、密钥、学习节奏、安全边界和提示词档位。

### 必须存在的功能

- DeepSeek 配置：
  - Base URL。
  - Model。
  - API Key。
  - API Key 加密保存状态。
- 学习节奏：
  - 默认学习块分钟数。
  - 每日可用学习时间窗。
- 提示词档位：
  - foundation。
  - standard。
  - advanced。
  - exam。
  - recovery。
  - 查看当前版本。
  - 保存新版本。
- 桌面行为：
  - 开机自启开关。
  - 关闭窗口后驻留托盘说明。
- 隐私与监控边界：
  - 明确当前只记录前台应用名、窗口标题、切换事件。
  - 允许暂停或关闭监控。

### 暂时不做的功能

- 不做云同步设置。
- 不做账号系统。
- 不做多设备登录。
- 不做高级模型路由。
- 不做完整成本统计。
- 不做远程插件系统。

## 2. 页面之间的数据关系

## 2.1 今日与计划

- 今日读取当前日期的 confirmed plan 和 active session。
- 如果没有 confirmed plan，今日引导用户去计划页生成或确认计划。
- 计划页创建 draft plan。
- 计划页确认 draft 后，今日页显示该 confirmed plan 的当前块。
- 同一天只能有一个当前 confirmed plan。确认新计划时，旧 confirmed plan 必须进入 archived 或被明确替换。

## 2.2 计划与学习

- 计划页定义 daily plan blocks。
- 学习页只执行已确认计划中的 blocks。
- 学习页开始 block 后创建 study session。
- 学习页完成 block 后更新 block 状态。
- 任务是否完成需要单独判断，不能默认因一个 block 完成而完成整个 task。

## 2.3 学习与复盘

- 学习页产生 study sessions、focus events、skip logs、notes。
- 复盘页按日期读取对应 plan、blocks、sessions、skip logs 和必要的 focus summary。
- 复盘页不得混入其他日期 session。
- 复盘结果写入 ai_reviews 或后续专用 review 记录。

## 2.4 复盘与计划

- 复盘页生成 plan adjustment proposal。
- 用户确认后，计划页接收该 proposal 并形成新的 draft plan 或 plan version。
- 未确认建议不得修改 confirmed plan。

## 2.5 学习与知识

- 学习 session 的备注、问题、错误可以进入知识页。
- 知识记录应保留来源 session 或 task。
- 知识页记录不应反向自动改变任务或计划状态。

## 2.6 设置与所有页面

- 设置提供 AI provider、model、API Key、prompt profile、默认 blockMinutes、学习时间窗。
- 计划页使用 blockMinutes、study windows 和 prompt profile。
- 导入、计划、复盘调用 AI 前必须检查 API Key。
- 学习页使用监控设置和隐私边界。

## 3. 当前版本完整核心闭环

V1 核心闭环定义为：

1. 用户进入今日页。
2. 系统判断当前状态。
3. 若缺 API Key，用户进入设置页配置 DeepSeek。
4. 用户进入计划页，粘贴或创建学习目标/计划。
5. AI 解析为目标和任务。
6. 用户检查解析结果，选择进入今日计划的任务。
7. AI 生成今日草稿计划。
8. 用户检查草稿计划。
9. 用户确认草稿，形成今日 confirmed plan。
10. 用户回到今日页，看到当前推荐学习块。
11. 用户进入学习页开始学习块。
12. 系统记录 session 和允许范围内的前台应用事件。
13. 用户完成、部分完成、跳过或暂停学习块。
14. 系统更新 block/session 状态，但不自动错误完成整个 task。
15. 用户进入复盘页。
16. AI 基于当天数据生成复盘和调整建议。
17. 用户确认或拒绝调整建议。
18. 被确认的建议回到计划页，生成下一版草稿。
19. 重要备注、问题、错误或总结可进入知识页。

V1 是否完成，以这个闭环能否稳定走通为准，不以页面是否存在为准。

## 5. 删除、合并或移动现有功能的建议

## 5.1 删除或降级

- 删除或降级托盘“开始当前学习块”，除非它能真正读取当前推荐块并启动 session。
- 删除复盘页右侧只有说明没有操作的“人工确认”占位，改为真实确认/拒绝区后再展示。
- 删除“今日历史”里不准确的“版本 N”表达，改为“草稿生成时间”或真正接入 plan_versions。

## 5.2 合并

- 当前“学习工作台”的导入与生成能力应拆到“计划”页，而不是继续让今日页承担完整规划工作。
- 当前“当前专注块”应移动到“学习”页，今日页只保留当前块摘要和开始入口。
- 复盘触发入口应统一到“复盘”页，托盘只打开对应页面，不直接生成不可见结果。

## 5.3 移动

- Prompt profile 编辑从普通设置区中分离为设置页的高级区域，避免和 API Key、模型配置混淆。
- 每日学习时间窗从隐藏 app setting 移到设置页。
- 开机自启从隐藏 app setting 移到设置页。
- 导入解析结果从工作台隐式刷新移动到计划页的明确检查区。
- 任务状态直接编辑从任务清单中弱化，避免绕过计划和学习流程造成状态冲突。

## 5.4 保留但修正

- 保留“AI 草稿必须确认”的原则。
- 保留本地 SQLite 作为事实源。
- 保留 DeepSeek OpenAI-compatible provider 抽象。
- 保留 prompt profile versioning。
- 保留低侵入监控边界。
- 保留今日历史概念，但必须明确它是 daily plan 历史还是 plan_versions 历史。

## 6. V1 不做清单

V1 明确不做：

- 自动抓取 ChatGPT/Codex 网页。
- 手机监管。
- 截图、录屏、键盘记录、浏览器历史读取。
- 强制锁屏。
- 云同步。
- 多用户协作。
- 社交排名或游戏化系统。
- 完整 RAG。
- 复杂知识图谱。
- AI 自动覆盖确认计划。
- AI 自动删除或完成任务。

## 7. 范围结论

V1 的重点是把六个一级页面重新分工，让用户在每一步都知道下一步动作，并让数据状态保持可信：

- 今日：告诉用户现在做什么。
- 计划：把目标变成确认计划。
- 学习：执行当前块。
- 知识：保存可复用学习材料和记录。
- 复盘：评估执行并提出调整建议。
- 设置：配置模型、节奏和隐私边界。

后续开发应优先修复核心闭环的数据语义，再做视觉精修或 Future 功能。
